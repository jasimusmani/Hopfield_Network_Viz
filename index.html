Memory from Scratch

How a physicist in 1982 taught a grid of ±1 numbers to remember — and how your brain might do the same thing.

📅 2026 ✍️ Jasim Usmani ⏱ 15 min read 🔬 From first principles
Start Reading ↓

Imagine you find an old photo torn into four pieces. Even with a quarter of the image missing, your brain fills in the rest — you recognize the face, the room, the moment. You do this instantly and effortlessly.

In 1982, physicist John Hopfield asked: could a mathematical model do the same thing? Could a network of simple binary nodes — each either firing or not — learn to store memories and recall them from partial or noisy inputs? The answer was yes. The Hopfield network was born, and it remains one of the most beautiful bridges between physics, neuroscience, and modern AI.

🏆
John Hopfield was awarded the 2024 Nobel Prize in Physics for this work — a recognition of how deeply physical intuition shaped our understanding of neural networks.

01The Intuition — Neurons as Light Switches

Start with the simplest possible neuron: a binary switch. It can be on (+1) or off (−1). Nothing else. Arrange $N$ of these neurons in a network, where every neuron is connected to every other.

Each neuron's state $s_i \in \{-1, +1\}$. The full state of the network at any moment is a vector:

Network State
$$\mathbf{s} = (s_1, s_2, \ldots, s_N) \in \{-1, +1\}^N$$

For a 12×10 pixel grid (like in our app), $N = 120$. Each pixel is one neuron: white = +1, black = −1. A "memory" is just one particular configuration of all 120 switches.

The question Hopfield answered is: given the noisy pattern on the right, how do we configure the weights between neurons so that running the network settles back into the clean pattern on the left?

02Teaching the Network — Hebbian Learning

To store $P$ patterns $\{\boldsymbol{\xi}^1, \boldsymbol{\xi}^2, \ldots, \boldsymbol{\xi}^P\}$, we need to set the weight $W_{ij}$ between every pair of neurons $i$ and $j$. The rule is beautifully simple — it's the famous Hebb rule: "neurons that fire together, wire together."

Hebbian Weight Matrix — the core learning rule
$$W_{ij} = \frac{1}{P} \sum_{\mu=1}^{P} \xi_i^{\mu} \, \xi_j^{\mu}, \qquad W_{ii} = 0$$

Where:

  • $\xi_i^{\mu}$ is the state of neuron $i$ in memory $\mu$ (either +1 or −1)
  • $P$ is the number of patterns stored
  • $W_{ij} = W_{ji}$ — the matrix is symmetric (no direction, just correlation)
  • $W_{ii} = 0$ — neurons don't connect to themselves
💡
What does this mean physically? If neurons $i$ and $j$ tend to be in the same state across all stored patterns (both +1 or both −1), their weight $W_{ij}$ will be positive — they reinforce each other. If they tend to be opposite, $W_{ij}$ is negative — they suppress each other. The network literally encodes the correlations between pixel positions.

The Weight Matrix in Python

Here's exactly how the app computes it — notice it's just a double loop over all neuron pairs:

def create_weight_matrix(input_arrays):
    # input_arrays shape: (P=8 patterns, N=120 neurons)
    weights = np.zeros((120, 120))

    for i in range(120):
        for j in range(120):
            if i == j:
                weights[i, j] = 0          # no self-connections
            else:
                w = 0
                for l in range(len(input_arrays)):
                    w += input_arrays[l, i] * input_arrays[l, j]  # ξᵢᵘ · ξⱼᵘ
                weights[i, j] = w / input_arrays.shape[0]         # divide by P
                weights[j, i] = weights[i, j]                     # symmetry

    return weights  # shape (120, 120)

This produces a 120×120 symmetric matrix of floating-point weights. In vectorized NumPy this is just: W = (X.T @ X) / P with the diagonal zeroed — but the loop makes the formula transparent.

⚠️
Memory is implicit. The patterns are never stored explicitly. All the information about all 8 patterns is compressed into one 120×120 matrix of numbers. This is what makes Hopfield networks remarkable — and why they can generalize from partial inputs.

03Recalling Memories — The Update Rule

We have a weight matrix. Now we present a noisy pattern and let the network evolve. Each neuron looks at its inputs from all other neurons, computes a weighted sum, and decides whether to flip to +1 or −1:

Neuron Update Rule
$$s_i(t+1) = \text{sign}\!\left(\sum_{j \neq i} W_{ij} \, s_j(t)\right) = \text{sign}\!\left(\mathbf{W}_i \cdot \mathbf{s}(t)\right)$$

The $\text{sign}$ function maps any positive number to +1 and any negative number to −1. If the weighted sum is exactly zero, the neuron stays unchanged.

Two Flavours of Update

ModeWhat happens each stepMathematical property
Synchronous All 120 neurons update simultaneously using the old state Guaranteed to converge in cycles of length ≤ 2
Asynchronous One random neuron updates at a time Guaranteed to monotonically decrease energy; always converges
🔬
Asynchronous update is the biologically plausible mode — real neurons don't all fire at exactly the same moment. Synchronous update is computationally convenient and mirrors how certain physical spin systems evolve.

Synchronous Update in One Line

def synchronous_update(pattern, weights):
    # W · s: matrix-vector multiply, then sign
    return np.sign(np.dot(weights, pattern))
    # shape: (120,) → (120,)

Asynchronous Update — One Neuron at a Time

def asynchronous_update(pattern, weights, max_iterations):
    n = len(pattern)
    updated_pattern = pattern.copy()

    for _ in range(max_iterations):
        i = random.randint(0, n - 1)          # pick a random neuron
        activation = np.dot(weights[i], updated_pattern)  # Σⱼ Wᵢⱼ sⱼ
        # apply threshold: +1 if positive, -1 if negative, unchanged if zero
        new_state = 1 if activation > 0 else -1 if activation < 0 else updated_pattern[i]
        updated_pattern[i] = new_state        # update in place

    return updated_pattern
Pick a pattern. Corrupt it: flip some pixels randomly according to the noise level.
Feed the noisy pattern into the update rule. Each neuron "votes" based on the weighted sum of its neighbours.
Neurons that agree with the stored memory get reinforced; neurons that disagree get flipped.
Repeat until the pattern stops changing — the network has settled at a fixed point (attractor).

04The Energy Landscape — Why It Always Converges

Here's the deepest insight: Hopfield networks are a physical system. Hopfield imported the concept of energy from statistical mechanics — specifically, the Ising model of magnetic spins. Every state $\mathbf{s}$ has an associated energy:

Hopfield Energy Function (Lyapunov Function)
$$E(\mathbf{s}) = -\frac{1}{2} \sum_{i \neq j} W_{ij} \, s_i \, s_j = -\frac{1}{2} \mathbf{s}^\top \mathbf{W} \mathbf{s}$$

The minus sign is important. When neurons aligned with the stored pattern are active, the positive weights between them contribute to a lower (more negative) energy. So stored patterns are energy minima — valleys in a landscape.

The energy landscape. Each green valley is a stored memory. The red ball (noisy input) rolls downhill and settles in the nearest valley.

Energy Always Decreases — The Proof

For asynchronous update, we can show that flipping neuron $i$ never increases energy. The change in energy when neuron $i$ flips from $s_i$ to $s_i'$ is:

Energy Change on Single Neuron Flip
$$\Delta E = -\Delta s_i \sum_{j \neq i} W_{ij} s_j$$

where $\Delta s_i = s_i' - s_i$. The update rule sets $s_i' = \text{sign}(\sum_j W_{ij} s_j)$, which means $\Delta s_i$ and $\sum_j W_{ij} s_j$ always have the same sign. Therefore $\Delta E \leq 0$ — energy never increases.

Since the energy is bounded below (the state space is finite — there are only $2^N$ possible states) and never increases, the network must converge to a fixed point. This is the key guarantee.

Watching Energy Fall in the App

def calculate_energy_evolution(pattern, weights, update_mode, iterations):
    energy_list = []
    current_pattern = pattern.copy()

    for _ in range(iterations):
        # E = -½ sᵀWs
        energy = -0.5 * np.dot(np.dot(current_pattern, weights), current_pattern)
        energy_list.append(energy)

        if update_mode == "Synchronous":
            current_pattern = synchronous_update(current_pattern, weights)
        else:
            current_pattern = asynchronous_update(current_pattern, weights, 1)

    return energy_list  # a list of floats, always non-increasing (async mode)

05Attractors — What Actually Gets Remembered

The fixed points of the update rule — the energy minima — are called attractors. Ideally, each stored pattern $\boldsymbol{\xi}^\mu$ is an attractor. But in practice, the network also creates spurious attractors: states that are energy minima but don't correspond to any stored pattern.

🕳️
Spurious memories. The most common spurious states are mixtures of stored patterns — typically linear combinations of an odd number of memories. For example, $\frac{1}{3}(\boldsymbol{\xi}^1 - \boldsymbol{\xi}^2 + \boldsymbol{\xi}^3)$ can be a spurious attractor. The network "misremembers."

Also worth noting: if pattern $\boldsymbol{\xi}^\mu$ is an attractor, so is $-\boldsymbol{\xi}^\mu$ (its complement) — because flipping all signs leaves the energy unchanged. In the app, this means the inverse of a digit (black/white swapped) is also a valid memory.

06Storage Capacity — How Much Can It Remember?

You can't store infinitely many patterns. As you add more, they start interfering with each other — the cross-correlation terms in the weight matrix create "crosstalk" noise that corrupts retrieval.

The critical result, derived by Amit, Gutfreund, and Sompolinsky using statistical physics in 1985, is that a Hopfield network with $N$ neurons can reliably store at most:

Storage Capacity (Amit–Gutfreund–Sompolinsky 1985)
$$P_{\max} \approx 0.138 \times N$$

Below this limit, retrieval accuracy stays high. Above it, the network degrades rapidly — like writing too many things on the same whiteboard until everything is illegible.

Our app uses $N = 120$ neurons and stores $P = 8$ patterns. The capacity limit is $0.138 \times 120 \approx 16.6$ patterns — so we're at about 48% capacity, well within the reliable zone.

Why 0.138?

The derivation uses the replica method from statistical mechanics. The key insight is that each stored pattern contributes "signal" of magnitude 1, while cross-pattern interference contributes noise of standard deviation $\sqrt{P/N}$. When $P/N \approx 0.138$, the signal-to-noise ratio drops below the threshold for reliable recall.

Signal vs Noise in Pattern Retrieval
$$h_i = \underbrace{\xi_i^1}_{\text{signal}} + \underbrace{\frac{1}{N}\sum_{\mu \neq 1} \sum_{j \neq i} \xi_i^\mu \xi_j^\mu s_j}_{\text{crosstalk noise}}$$

The crosstalk noise has variance $\approx P/N$. Reliable recall requires this to be small.

07Noise Tolerance — How Broken Can the Input Be?

Even within the capacity limit, retrieval degrades as you add more noise. There's a phase transition: below a critical noise level $m_c$, the network recovers the correct pattern. Above it, retrieval fails.

Noise Level in the App
$$\text{Noise level} = \frac{\text{# flipped pixels}}{N} \in [0, 1]$$

In the app, noise level 0.25 means 25% of the 120 pixels are randomly flipped. That's 30 pixels changed — yet the network typically recovers the pattern perfectly, because the basin of attraction of each memory is large.

🎯
Basins of attraction. Each stored memory has a "basin" — the set of all states that will converge to it. The size of each basin depends on how similar the stored patterns are to each other. Orthogonal patterns (no overlap) have the largest, most symmetric basins.

2. Adding Noise

To simulate a degraded memory, we randomly flip a fraction of pixels. noise_level=0.25 flips $0.25 \times 120 = 30$ pixels. 

def add_noise(pattern, noise_level):
    noisy_pattern = pattern.copy()
    num_pixels = int(120 * noise_level)      # how many to flip
    for _ in range(num_pixels):
        pixel = random.randint(0, 119)
        noisy_pattern[pixel] *= -1           # flip: +1 → -1 or -1 → +1
    return noisy_pattern

09From Hopfield to Transformers

Hopfield networks aren't just historical curiosities. In 2020, Ramsauer et al. showed that modern Hopfield networks — with a softmax-based energy function instead of the sign rule — have exponentially higher storage capacity and that their update rule is mathematically identical to the attention mechanism in Transformers.

Modern Hopfield Update = Attention (Ramsauer et al. 2020)
$$\mathbf{s}^{\text{new}} = \mathbf{X} \cdot \text{softmax}\!\left(\beta \mathbf{X}^\top \mathbf{s}\right)$$

where $\mathbf{X}$ is the matrix of stored patterns (keys/values) and $\beta$ is an inverse temperature. This is precisely scaled dot-product attention — the heart of GPT, BERT, and every modern LLM.

🤯
The punchline: every time a Transformer attends to its context window, it is performing one step of a modern Hopfield network retrieving a stored pattern. The 1982 physics model and the 2017 deep-learning revolution are the same thing, viewed from different angles.

10What We Built, and What It Means

Let's recap the whole story:

Represent patterns as vectors of ±1 values — one entry per pixel, one vector per memory.
Encode memories into weights using Hebbian learning: $W_{ij} = \tfrac{1}{P}\sum_\mu \xi_i^\mu \xi_j^\mu$
Present a noisy query. Run the update rule until convergence: $s_i \leftarrow \text{sign}(\mathbf{W}_i \cdot \mathbf{s})$
The energy descends monotonically to a minimum — a stored memory. Pattern retrieved.

With 120 neurons and 8 patterns, this tiny network reliably reconstructs corrupted images — not by explicitly looking up a table, but by evolving a physical system toward its lowest energy state. That's the magic.

And sitting underneath the attention heads of the largest language models in the world is exactly this mechanism — just scaled, softened, and given a different name. Hopfield's 1982 paper is, in a very real sense, the foundation stone of modern AI.
